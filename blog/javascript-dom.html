Javascript's relationship with HTML and CSS

The relationship between these three amigo's requires some knowledge of all. JavaScript would not even know the HTML and CSS exists without the DOM. The DOM is made up of Nodes and Objects, these nodes and objects have elements. As I've mentioned above, the DOM enables JavaScript to manipulate elements within the HTML and CSS files to modify, change and delete elements by accessing the DOM's nodes and objects.(sorry a bit overkill with 'the DOM' but sounds like 'the DON' a mafia boss 不不不).

How does JavaScript access elements within a webpage?

By creating a function. A Function is a set of statements that performs a task(like changing the font-size) or calculating a value (like adding some numbers). To qualify as a function it needs a beginning, middle and end. A name, a list of parameters and what you want out of it.

Example: If I wanted to know how many AC/DC songs I have in my playlist(that's a long list).How would I access my playlist to find out how many AC/DC songs I have in it? I could go through and physically count each song(hell no...too many) or create a JavaScript function to do it for me. How would JavaScript know to count them all and not just one? I would create an array, which would group all my AC/DC songs together.(Thank goodness I don't need to do that). Which brings me to the next bit of information.

Arrays and Objects. Arrays will group entities with same type e.g AC/DC being type and the songs being the items attached. Objects are single entities with named properties. e.g a container is an object that has properties of height, width, color etc.
